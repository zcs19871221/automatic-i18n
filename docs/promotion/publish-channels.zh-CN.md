# automatic-i18n 发布渠道与投放顺序

这份清单按优先级排列，目标是低成本拿到首批真实用户反馈。

## 第一梯队：优先投放

1. GitHub README 顶部

- 放一段 20-40 秒 GIF
- 标题下第一屏写一句价值主张
- 提供一键命令和前后对比

2. 掘金

- 适合中文前端开发者
- 文章标题建议：老项目国际化改造太慢，我做了一个一键替换工具

3. CSDN

- 流量稳定，适合工具型教程
- 可复用掘金文章，调整语气为教程风

4. 公众号或个人博客

- 适合沉淀完整故事：问题背景、方案、效果

## 第二梯队：英文用户获取

1. Dev.to

- 英文技术读者友好
- 标题建议：One command to migrate hard-coded UI text to i18n

2. Reddit

- 子版块建议：r/reactjs、r/typescript、r/webdev
- 发帖要以问题和方案为主，避免纯推广

3. X（Twitter）

- 发短帖，附 GIF 和仓库链接
- 适合拉第一波转发

## 第三梯队：工具目录和长期曝光

1. Product Hunt

- 适合准备完整英文落地页后发

2. Awesome 列表

- 提交到 i18n、frontend、typescript 相关列表

3. npm 页面

- 保持描述和 README 第一屏一致

## 推荐发布时间

- 中文平台：工作日 10:00-12:00 或 19:00-22:00
- 英文平台：北京时间 21:00-次日 1:00

## 一周执行节奏

Day 1

- 完成 GIF、README 首屏优化

Day 2

- 发布掘金文章 + 同步 CSDN

Day 3

- 发布 Dev.to 英文版

Day 4

- 在 Reddit 发场景帖

Day 5

- 整理反馈，修正文档与 FAQ

Day 6

- 发一条更新动态：已解决的问题和新能力

Day 7

- 汇总数据：Star、Issue、安装量、文章阅读

## 文章分发最小包

每次发文至少带这三样：

- 一句话价值主张
- 20-40 秒演示 GIF
- 一条可执行命令
