# English Promotion Channels and Rollout Plan

This plan is optimized for early traction with low marketing budget.

## Tier 1: High-Impact Channels

1. GitHub README (Top Priority)

- Add a 20-40 second demo GIF above the fold
- Put one-sentence value proposition in the first screen
- Show before/after + one command

2. Dev.to

- Great for frontend/tooling audience
- Suggested title: One command to migrate hard-coded UI text to i18n

3. X (Twitter)

- Post short thread with GIF + repo link
- Pin the post for 1-2 weeks

4. Reddit

- Recommended subreddits: r/reactjs, r/typescript, r/webdev
- Frame as problem + solution, avoid pure self-promo tone

## Tier 2: Long-Tail Discovery

1. Product Hunt

- Best after polishing landing narrative and visuals

2. Awesome Lists

- Submit to i18n/frontend/typescript/tooling lists

3. npm Page

- Keep npm description aligned with README first screen

## Suggested Posting Time (UTC)

- Dev.to: 12:00-16:00 UTC
- X: 13:00-17:00 UTC or 00:00-02:00 UTC
- Reddit: 14:00-18:00 UTC

## 7-Day Rollout

Day 1

- Finalize GIF and README first screen

Day 2

- Publish Dev.to long-form post

Day 3

- Post X thread with before/after and command

Day 4

- Publish Reddit post with practical migration story

Day 5

- Collect comments/issues and update FAQ

Day 6

- Share a progress update: fixes and roadmap

Day 7

- Measure impact: stars, npm downloads, issues, article views

## Minimum Distribution Pack

Each promotion post should include:

- One-sentence value proposition
- 20-40 second GIF
- One runnable command
- Repo + npm links
